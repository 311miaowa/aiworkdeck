# WPS 替换功能 - 快速测试用例

## 测试准备

### 1. 创建测试文档
打开WPS编辑器，创建新文档，复制以下内容：

```
甲方和乙方签订了合同。甲方应支付货款，乙方应交付货物。
如果甲方违约，甲方应承担违约责任。如果乙方违约，乙方也应承担责任。
本合同由甲方和乙方共同签署。
```

保存为 `替换测试.docx`

### 2. 打开浏览器控制台
- 按 `F12` 打开开发者工具
- 切换到 **Console** 标签
- 确保能看到 `[WpsBridge]` 开头的日志

## 常见问题

### Q1: 替换后文档没有变化
**检查**：
1. 控制台是否有 `[WpsBridge] executeDelete failed` 错误
2. 日志中选区是否正确（如 `70-72` 而不是 `70-70`）
3. 是否有 `[WpsBridge] Replacement success: true`

### Q2: 只替换了部分
**检查**：
1. 查看日志中有几次 `[WpsBridge] replaceAtPosition`
2. 每次替换后是否成功

### Q3: AI说成功了但实际没变
**原因**：可能是 `range.Text = ""` 不生效，但没有使用 Selection.Delete() 备用方案
**检查**：日志中是否有 `Selection.Delete() completed`

## 修复历史

### 修复1: Selection设置方式（2025-01-11）
- ❌ 错误：`sel.Start = start; sel.End = end` → 选区变成 `70-70`
- ✅ 正确：`await selRange.SetRange(start, end)` → 选区正确 `70-72`

### 修复2: Delete参数（2025-01-11）
- ❌ 错误：`sel.Delete(1)` → 只删除1个字符
- ✅ 正确：`sel.Delete()` → 删除整个选区

### 修复3: 关闭修订模式（2025-01-11）
- ✅ 添加 `disableTrackRevisions()` 在所有AI操作前执行
