# WPS 文档替换功能测试用例

## 测试目的
验证 AI 助手能够正确理解和执行多种替换方式：
1. 全部替换
2. 替换第一个匹配项
3. 替换第 N 个指定的匹配项
4. 删除操作（全部删除、删除第 N 个）

## 测试环境准备

### 测试文档内容
创建一个测试文档 `替换测试.docx`，内容如下：

```
甲方和乙方签订了合同。甲方应支付货款，乙方应交付货物。
如果甲方违约，甲方应承担违约责任。如果乙方违约，乙方也应承担责任。
本合同由甲方和乙方共同签署。
```

## 测试用例

### TC-001: 全部替换功能

**测试描述**: 验证AI能够正确执行全部替换操作

**用户输入**: "把所有的'甲方'改成'买方'"

**预期AI行为**:
1. 思考过程：识别用户要求全部替换
2. 调用工具：`wps_find_replace(findText="甲方", replaceText="买方", replaceAll=true)`
3. 预期结果：
   - 所有3处"甲方"都被替换为"买方"
   - 文档显示为修订模式，用户可以看到变更

**预期文档结果**:
```
买方和乙方签订了合同。买方应支付货款，乙方应交付货物。
如果买方违约，买方应承担违约责任。如果乙方违约，乙方也应承担责任。
本合同由买方和乙方共同签署。
```

**通过标准**:
- [ ] AI调用了 `wps_find_replace` 且 `replaceAll=true`
- [ ] 所有"甲方"都被替换
- [ ] 替换以修订模式显示

### TC-003: 替换第 N 个指定的匹配项

**测试描述**: 验证AI能够精确定位并替换第 N 个匹配项

**用户输入**: "把第2个'甲方'改成'申请人'"

**预期AI行为**:
1. 思考过程：识别用户要求替换第2个匹配项
2. 调用工具：`wps_replace_nth_match(findText="甲方", replaceText="申请人", matchIndex=2)`
3. 预期结果：第2个"甲方"被替换

**预期文档结果**:
```
甲方和乙方签订了合同。甲方应支付货款，乙方应交付货物。
如果申请人违约，申请人应承担违约责任。如果乙方违约，乙方也应承担责任。
本合同由甲方和乙方共同签署。
```

**通过标准**:
- [ ] AI调用了 `wps_replace_nth_match` 且 `matchIndex=2`
- [ ] 只有第2个"甲方"被替换（第2段开头的）
- [ ] 其他"甲方"保持不变

---

### TC-004: 替换倒数第 N 个匹配项

**测试描述**: 验证AI能够处理"倒数第 N 个"的复杂指令

**用户输入**: "把倒数第1个'违约'改成'毁约'"

**预期AI行为**:
1. 思考过程：识别用户要求替换倒数第1个
2. 调用工具：
   - 先调用 `wps_find_text(keyword="违约")` 查找所有匹配位置
   - 确定总数为2，倒数第1个即第2个
   - 调用 `wps_replace_nth_match(findText="违约", replaceText="毁约", matchIndex=2)`
3. 预期结果：最后1个"违约"被替换

**预期文档结果**:
```
甲方和乙方签订了合同。甲方应支付货款，乙方应交付货物。
如果甲方违约，甲方应承担违约责任。如果乙方毁约，乙方也应承担责任。
```

**通过标准**:
- [ ] AI先调用了 `wps_find_text` 查找位置
- [ ] AI正确计算出倒数第1个对应的索引
- [ ] AI调用了 `wps_replace_nth_match` 且索引正确

### TC-008: 复杂替换场景 - 同时替换多个词

**测试描述**: 验证AI能够处理连续的多个替换请求

**用户输入**: "把'甲方'全部改成'买方'，然后把'乙方'全部改成'卖方'"

**预期AI行为**:
1. 思考过程：识别用户有两个连续的替换请求
2. 第一次调用：`wps_find_replace(findText="甲方", replaceText="买方", replaceAll=true)`
3. 等待结果后，第二次调用：`wps_find_replace(findText="乙方", replaceText="卖方", replaceAll=true)`
4. 预期结果：两处替换都完成

**预期文档结果**:
```
买方和卖方签订了合同。买方应支付货款，卖方应交付货物。
如果买方违约，买方应承担违约责任。如果卖方违约，卖方也应承担责任。
本合同由买方和卖方共同签署。
```

**通过标准**:
- [ ] AI依次执行了两次替换操作
- [ ] 所有"甲方"被替换为"买方"
- [ ] 所有"乙方"被替换为"卖方"

---

**用户输入**: "把'丙方'改成'丁方'"

2. 调用工具：`wps_find_replace(findText="丙方", replaceText="丁方", replaceAll=true)`

---

### TC-010: 边界情况 - matchIndex 超出范围

**测试描述**: 验证AI能够处理超出范围的索引请求

**用户输入**: "把第10个'甲方'改成'买方'"

**预期AI行为**:
1. 思考过程：先查找确认匹配项数量
2. 调用工具：
   - 先调用 `wps_find_text(keyword="甲方")` 查找所有匹配
   - 发现只有3个匹配项
   - 告知用户只有3个"甲方"，无法替换第10个
3. 预期结果：AI向用户说明情况

**通过标准**:
- [ ] AI先调用了 `wps_find_text` 确认数量
- [ ] AI正确告知用户匹配项不足
- [ ] AI没有执行无效的替换操作
